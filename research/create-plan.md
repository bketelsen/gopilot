# Create a plan from an issue

Idea: create an issue in the repository with an indicator (gopilot:plan or similar) that invokes a new agent that goes through interactive planning INSIDE the github issue, asking pertinent questions, refining the plan, in the same way that superpowers:brainstorm does. When the plan is complete and approved, it's converted into GH project & issues that gopilot expects with dependencies, etc all worked through.

If creating an issue doesn't make sense, then some sort of interactive cli experience would work as well. a cli command, or a tool/skill/agent in copilot.
